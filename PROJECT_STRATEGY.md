# ПРОЕКТ: ЦИФРОВОЙ КОД (DIGITAL CODE)
**Стратегия разработки, архитектура и методология продукта**

## 1. МЕТОДОЛОГИЧЕСКИЙ ФУНДАМЕНТ (СИНТЕЗ)
Продукт строится на стыке четырех дисциплин, переведенных на безопасный, неклинический язык (без терминов "терапия", "диагноз", "пациент").

### 1.1. Образное моделирование (на базе ЭОТ)
**Суть:** Работа с внутренними блоками через визуальные метафоры и телесный отклик.
**Интеграция в систему:** 
- **Техника "Распаковка узла":** Если у пользователя конфликт чисел (например, Душа 2, Путь 8), система не просто констатирует факт, а выдает практику. 
- *Шаг 1 (Тело):* "Где в теле ты чувствуешь тяжесть восьмерки?"
- *Шаг 2 (Образ):* "Представь эту тяжесть как предмет."
- *Шаг 3 (Трансформация):* "Мысленно передай этому предмету свою благодарность и позволь ему изменить форму."
Это дает мгновенное облегчение и инсайт.

### 1.2. Внутренний Театр (на базе Юнгианских архетипов и Самости)
**Суть:** Интеграция теневых аспектов и поиск Истинного Я (Самости).
**Интеграция в систему:**
- 5 Главных Чисел — это 5 субличностей (архетипов) во Внутреннем Театре.
- **Число Души** — это ядро (Самость).
- **Число Выражения** — это Персона (социальная маска).
- **Пустые зоны матрицы** — это Тень (то, что вытеснено, но таит огромный ресурс).
Система помогает пользователю "познакомиться" со своими фигурами и посадить их за один стол переговоров.

### 1.3. Метафорический нарратив (на базе Сказкотерапии)
**Суть:** Обход логического сопротивления ума через метафору.
**Интеграция в систему:** Персональная сказка, генерируемая нейросетью на основе 5 чисел. Сказка содержит "зашифрованный" ключ к решению текущей жизненной задачи пользователя.

### 1.4. Символический набросок (Sketch-практики)
**Суть:** Проекция бессознательного на бумагу.
**Интеграция в систему:** В конце разбора пользователь получает задание нарисовать свой "Финальный образ" (даже схематично) или собрать его в воображении. Это закрепляет нейронную связь.

## 2. ПРОДУКТОВОЕ РЕШЕНИЕ И ФОРМ-ФАКТОР: "ЗЕРКАЛО"

**Концепция:** Цифровое "Зеркало", которое использует древние знания и новые технологии, чтобы раскрыть человека для него самого.

**Эстетика (UI/UX):** Древнегреческий стиль. Чистый, светлый, очень красивый. 
- **Цвета:** Белый мрамор (Alabaster), золото (Gold), глубокий графит для текста.
- **Типографика:** Элегантные антиквы (Serif) для заголовков, чистые гротески (Sans) для текста.
- **Формы:** Строгая геометрия, симметрия, золотое сечение, много "воздуха" и света.

**Экосистема Агентов (Agentic Team):**
Проект обслуживается командой специализированных ИИ-агентов:
1. **YouTube Avatar Agent (Медийный двигатель):** Создает ценный контент, генерирует видео с аватаром, проводит разборы известных личностей, привлекая трафик.
2. **Calculation Agent (Бэкенд):** Отвечает за безупречную математическую точность (TypeScript ядро).
3. **Editorial Agent (Альберт Вяземский):** Генерирует глубокие, литературные разборы на основе расчетов.

## 3. ТЕХНИЧЕСКАЯ АРХИТЕКТУРА

### 3.1. Frontend (Клиентская часть)
- **Стек:** React 19 + Vite + Tailwind CSS + Framer Motion (для магии и плавных анимаций).
- **UI/UX:** Минималистичный, глубокий дизайн. Темные тона, стекло (glassmorphism), тонкая типографика. Интерфейс должен ощущаться как дорогой, мистический, но научный инструмент.

### 3.2. Backend (Серверная часть)
- **Стек:** Node.js + Express.
- **База данных:** Firebase Firestore (хранение профилей пользователей, их матриц, истории разборов и совместимостей).
- **Аутентификация:** Firebase Auth (Google, Telegram Login).

### 3.3. AI Core (Нейро-движок)
- **Стек:** Google Gemini API.
- **Логика:** Backend принимает дату рождения -> делает точный математический расчет (TypeScript) -> формирует промпт на основе `AGENTS.md` и 13 Скиллов -> отправляет в Gemini -> получает литературный, глубокий разбор -> сохраняет в БД -> отдает на Frontend.
