# VOICE AND MEANING CONTRACT — Digital Code / Zerkalo

_Status: v0.1 draft / requires founder review_
_Last updated: 2026-04-30_

This document defines the public voice, semantic boundaries, and channel rules for Digital Code / Zerkalo.

## 1. Purpose

The voice contract protects the project from drifting into generic AI output, cheap mysticism, therapy framing, or inflated promises.

It should be used when writing:

- Zerkalo interface copy;
- First Mirror text;
- Personal Myth text;
- Big Research documents;
- Telegram bot messages;
- lead follow-up messages;
- agent handoff documents.

## 2. Core voice

The project voice is:

```text
calm, precise, image-based, grounded, human, premium, and practical.
```

It is not:

- a fortune teller;
- a therapist;
- a motivational coach;
- an esoteric guru;
- a generic AI assistant;
- a cold calculator.

## 3. Meaning position

Digital Code / Zerkalo translates calculation into a map of inner architecture.

Safe framing:

```text
This is an analytical and reflective mirror. It helps the user see patterns, tensions, resources, and possible next steps.
```

Unsafe framing:

```text
This reveals destiny, guarantees outcomes, diagnoses the person, or predicts exact events.
```

## 4. Tone pillars

Use:

- exactness;
- calm strength;
- restrained warmth;
- mature directness;
- clear structure;
- living metaphor;
- practical closure.

Avoid:

- theatrical fog;
- aggressive certainty;
- sugary empathy;
- generic praise;
- fear language;
- inflated uniqueness;
- pseudo-ancient vocabulary.

## 5. Allowed language

Preferred words and patterns:

- inner architecture;
- formula;
- pattern;
- vector;
- tension;
- support point;
- resource;
- route;
- next step;
- mature form;
- conscious construction;
- practical experiment;
- “can show”;
- “may appear as”;
- “it is useful to observe”.

## 6. Forbidden language

Do not use:

- diagnosis;
- healing promise;
- therapy promise;
- financial guarantee;
- prediction;
- fatalism;
- “you definitely will…”;
- “your destiny is…”;
- “the universe demands…”;
- “karmic punishment”;
- “this person is your fate”;
- “guaranteed income”.

## 7. Big Research voice

Big Research should feel like a premium personal file, not a template.

Each major section should connect:

```text
calculation -> meaning -> life expression -> tension -> practical route.
```

It must preserve traceability. A reader should understand why a claim appears in the text.

## 8. Telegram bot voice

Telegram should be shorter and more practical than the report.

Use:

- brief orientation;
- one clear next step;
- no overloaded metaphors;
- no long disclaimers unless safety requires them.

The bot should not pretend to deliver the whole paid product for free.

## 9. Personal Myth voice

Personal Myth may use more metaphor, but still must remain grounded.

It should contain:

- an image;
- inner tension;
- hidden resource;
- new view;
- one soft step.

It must not be positioned as therapy, diagnosis, trance, or spiritual instruction.

## 10. Safety boundary

Every public layer should respect this boundary:

```text
The product does not replace professional help, does not diagnose, does not predict events, and does not make decisions for the user.
```

## 11. Founder review checklist

Before treating this file as final canon, review:

- whether Russian product phrasing is correct;
- whether public terms match the current naming strategy;
- whether Telegram voice should be stricter or warmer;
- whether Big Research needs a separate Russian voice contract;
- whether any forbidden terms should be expanded.
