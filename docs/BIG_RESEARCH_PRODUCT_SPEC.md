# BIG RESEARCH PRODUCT SPEC — Digital Code / Zerkalo

_Last updated: 2026-04-29_

This document defines the paid product: Big Research / Full Mirror.

## 1. Product definition

Big Research is the main paid product of Digital Code / Zerkalo.

It is a premium personal research file based on:

- date calculation;
- five main numbers;
- compound numbers;
- matrix;
- current period / personal year when available;
- user context when provided;
- living interpretation over a strict calculation base.

It is not an automatic horoscope, not a therapy document, not a prediction, and not a generic AI report.

## 2. MVP format

Recommended MVP size: **15–30 pages**.

Do not chase 80 pages before demand is validated. The first paid version should be:

- exact;
- structured;
- visually premium;
- personal;
- useful enough to justify payment;
- fast enough to prepare manually for first clients.

## 3. Core promise

```text
A precise personal file that turns the calculation into a clear map of strengths, tensions, realization routes, and the next practical step.
```

## 4. Product boundary

The document must state:

- not a diagnosis;
- not therapy;
- not medical, legal, financial advice;
- not a prediction;
- not a guarantee;
- an analytical and reflective document.

## 5. Recommended section structure

### 1. Cover

Purpose: create premium first impression and orientation.

Must include:

- name;
- date of birth;
- formula;
- date of preparation;
- document title;
- short subtitle;
- author/project mark.

Avoid:

- mystical claims;
- overpromising;
- cheap visual effects.

### 2. How to read this document

Purpose: set the rules of perception.

Must explain:

- calculation layer;
- canon layer;
- living interpretation layer;
- the document is a mirror, not a verdict.

Quality benchmark: the Evgeniya report opening logic — calculation, canon, living interpretation.

### 3. Numeric passport

Purpose: give the exact skeleton.

Must include:

- Soul / Mind number;
- Path / Action number;
- Expression number if used;
- Direction / Realization number;
- Result / Outcome number;
- compound values;
- planets / symbolic energies;
- short formula sentence.

### 4. Main pattern

Purpose: synthesize the whole formula.

Must answer:

- what is the main inner mechanism;
- where force starts;
- how it moves;
- where it asks to mature;
- what the central life tension is.

### 5. Soul / Mind number

Purpose: describe inner core.

Must include:

- basic number;
- compound number if present;
- strength;
- tension;
- practical recommendation.

### 6. Path / Action number

Purpose: describe how the person acts.

Must include:

- action style;
- best use of force;
- common distortion;
- one concrete behavior shift.

### 7. Expression number

Purpose: describe how the person is perceived / enters the field.

Use only if it is part of the current calculation logic.

### 8. Direction / Realization number

Purpose: describe where the person should direct force.

Must include:

- realization vector;
- suitable domains;
- route to result;
- compound transformation if available.

### 9. Result / Outcome number

Purpose: describe mature assembly.

Must include:

- what the person is moving toward;
- what quality must be developed;
- what becomes central later;
- what to stop confusing with maturity.

### 10. Compound numbers

Purpose: show depth beyond the root digit.

Must include:

- formula of transition;
- first digit as starting energy;
- second digit as transformation / accent;
- root as final pattern;
- age/activation only if supported by canon.

### 11. Matrix of resources and tasks

Purpose: show what is present, repeated, absent, and trainable.

Must avoid calling gaps “defects”.

Correct framing:

```text
not empty = bad, but empty = conscious construction zone.
```

### 12. Strengths

Purpose: identify real support points.

Must be based on the formula and matrix, not generic traits.

### 13. Tension points

Purpose: show where the formula asks for form.

Do not use fear language. Use:

- tension;
- friction;
- overuse;
- undeveloped form;
- choice point.

### 14. Money and realization

Purpose: translate the formula into useful work and earning style.

Must include:

- where the person can create value;
- what work style suits them;
- where they lose money/energy;
- what to test in the next 30 days.

Do not guarantee income.

### 15. Relationships

Purpose: explain contact style.

Must include:

- what the person seeks in contact;
- what they may overdo;
- what stabilizes connection;
- what should be discussed consciously.

Do not promise love, marriage, breakup, or fate.

### 16. Personal year / current period

Use only if calculation is available.

Must include:

- year number;
- general theme;
- how it interacts with the main formula;
- what is useful to focus on.

Do not over-personalize period interpretation as canon if not supported.

### 17. 30-day practical route

Purpose: convert insight into action.

Must include:

- 3 support points;
- 3 risks / tension points;
- 3 actions;
- one main step.

### 18. Living Word

Purpose: closing address.

It should feel personal, not generic.

Good pattern:

- name the central truth;
- show the old pattern;
- show the new form;
- give one clear final line.

### 19. What to do next

Options:

- reread the main pattern;
- choose one 30-day step;
- ask one follow-up question;
- book live consultation;
- receive follow-up in Telegram.

## 6. Quality benchmark

A strong Big Research file should feel like:

- exact calculation;
- premium editorial document;
- living human recognition;
- practical route;
- no cheap mysticism;
- no vague AI-generated praise.

## 7. First sale version

For first paid smoke-test, use:

- manual lead;
- manual payment;
- manual or semi-manual document generation;
- human QA;
- delivery via Telegram or PDF link.

Automation comes only after 1–3 real paid or semi-paid cases.
