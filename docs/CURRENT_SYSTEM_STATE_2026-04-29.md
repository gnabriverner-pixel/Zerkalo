## 1. Что сейчас существует

Проект состоит из пяти контуров:

1. **Zerkalo web app** — визуальный вход, Первое зеркало, Личный Миф, заявка.
2. **Telegram-бот «Цифровой Код»** — канал сопровождения, DeepSeek-диалог, возврат пользователя, уведомления.

5. **VPS Beget** — production-сервер, где живут бот и Zerkalo как отдельные сервисы.

## 2. Zerkalo web app

Репозиторий:

```text
gnabriverner-pixel/Zerkalo
```

Публичные режимы:

1. `Архитектура`
2. `Личный Миф`

Скрытые / замороженные режимы:

- `Совместимость`
- отдельная `Сказка`

Они не должны возвращаться в публичный UI до проверки спроса на основной продукт.

### Что делает «Архитектура»

- принимает дату рождения;
- считает 5 главных чисел;
- показывает матрицу;
- показывает карточки чисел;
- генерирует Первое зеркало;
- предлагает заявку на Большое исследование.

### Что делает «Личный Миф»

- задаёт 4 вопроса о состоянии;
- генерирует образную историю через Gemini, если ключ доступен;
- при отсутствии Gemini показывает честный fallback-слой без имитации сказки;
- показывает главный образ, внутреннее напряжение, скрытый ресурс, новый взгляд;
- предлагает заявку на Большое исследование.

### Backend Zerkalo

Основные endpoints:

- `GET /health`
- `POST /api/generate`
- `POST /api/lead`

`/api/generate-pdf` может существовать как заглушка, но публичный UI не должен использовать его как реальную функцию.

### Lead flow

`/api/lead`:

1. принимает имя, дату рождения, контакт, запрос, source;
2. сохраняет заявку в `leads.json`;
3. если есть env `TELEGRAM_BOT_TOKEN` и `TELEGRAM_ADMIN_CHAT_ID`, отправляет владельцу уведомление в Telegram;
4. если Telegram-уведомление не удалось — пользователь всё равно получает success, а ошибка фиксируется только в developer log.

Статус:

- подключён DeepSeek;
- в `/start` добавлена кнопка `Открыть Зеркало`;

Роль бота:

- не заменять Zerkalo;
- не дублировать весь визуальный интерфейс;

7. Совместимость заморожена.
8. Личный кабинет не делается.
