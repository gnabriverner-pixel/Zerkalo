# PROJECT MEANING MAP — «Цифровой Код / Zerkalo»

> Версия: 1.0
> Дата: 2026-05-01
> Статус: Каноничный документ. Любые продуктовые решения сверяются с этой картой.

---

## 1. Что такое «Цифровой Код / Zerkalo»

Аналитико-образная система самопознания, которая через дату рождения выстраивает **персональную числовую архитектуру** — не гороскоп, не предсказание, не терапию, а точную структурную карту внутренних опор, напряжений и направлений.

Метод опирается на ведическую числовую традицию, но переведён на язык **практической аналитики**: конкретные механизмы, поведенческие паттерны, рабочие шаги.

---

## 2. Чем проект НЕ является

| Проект НЕ является | Почему |
|---|---|
| Нумерологией | Не используем мистическую рамку, не обещаем судьбу |
| Эзотерикой | Нет «энергий Вселенной», «кармических долгов», «вибраций» |
| Психотерапией | Не ставим диагнозов, не лечим, не заменяем специалиста |
| Предсказанием | Код показывает архитектуру, а не приговор |
| Гаданием | Нет случайных карт, нет интуитивных раскладов — только расчёт |
| Мотивационным коучингом | Не говорим «ты можешь всё» — говорим «вот как ты устроен» |

---

## 3. Главная ценность для пользователя

**Ясность собственной конструкции.**

Человек получает не абстрактное описание характера, а конкретную карту:
- Где его природная сила.
- Где привычная ловушка.
- Где точка выбора.
- Что делать в ближайшие 30 дней.

Ощущение после хорошего разбора: «Меня увидели. Это точно. Это про меня. Я понимаю, куда двигаться.»

## 4. Двери входа

### Дверь 1: Дата рождения
Ввод даты → расчёт 5 главных чисел → Первое зеркало.
Канал: Zerkalo web app, Telegram-бот.

### Дверь 2: Внутреннее состояние / образ
4 вопроса о текущем ощущении → Личный Миф (образная история).
Канал: Zerkalo web app (секция «Личный Миф»).

### Дверь 3: Telegram-диалог
/start → ввод имени, пола, даты → расчёт → Basic или Deep Code.
Канал: Telegram-бот.

---

## 5. Что есть и чем отличается

### Zerkalo (Web App)
**Роль:** Визуальный премиальный вход. Первое впечатление. Заявка.
**Что даёт бесплатно:**
- Первое зеркало (5 чисел, формула, 4 блока аналитики).
- Личный Миф (образная история по 4 вопросам).
**Что не даёт:** Полный разбор, матрицу, денежный код, 30-дневный план.
**CTA:** Заявка на Большое исследование.

### Telegram-бот
**Роль:** Рабочая среда. Расчёт, покупка, разбор, диалог, follow-up.
**Что даёт:**
- Basic Code (10 секций, 75 Stars).
- Deep Code (22 секции, PDF, Dialogue Mode, 190 Stars).
- Follow-up (day 3, 7, 14).
- Dialogue Mode (после Deep).
**CTA:** Покупка Basic/Deep через Stars.

### Личный Миф / Сказка про тебя
**Роль:** Образный, нерасчётный вход. Другая дверь к самопознанию.
**Что даёт:** Метафорическую историю на основе текущего состояния.
**Чего не даёт:** Числового расчёта, матрицы, прогноза.

### Большое исследование
**Роль:** Главный платный продукт. Полный персональный документ.
**Текущий статус:** Ручная заявка → ручная сборка → ручная отправка.
**Будущий статус:** Автоматический PDF через DeepSeek.

## 6. Как они соединяются

```
Пользователь
    ├─→ Zerkalo Web App
    │       ├── Архитектура Кода → Первое зеркало (бесплатно)
    │       ├── Личный Миф → Образная история (бесплатно)
    │       └── CTA → Заявка на Большое исследование → Telegram notification → Ручная продажа
    │
    └─→ Telegram-бот
            ├── /start → Расчёт → Basic/Deep Code (Stars)
            ├── Deep Code → 22 секции + PDF + Dialogue Mode
            ├── Follow-up → day 3, 7, 14
            ├── Кнопка «Открыть Зеркало» → Zerkalo Web App
            └── (Будущее) Доставка Большого исследования
```

## 11. Принципы, которые нельзя нарушать

1. **Не обещать судьбу.** Код — архитектура, не приговор.
2. **Не скатываться в эзотерику.** Никаких вибраций, карм, высших сил.
3. **Не ставить диагнозов.** Никаких медицинских, психологических, финансовых гарантий.
4. **Не давать скидок на Deep.** Premium не дисконтируется.
5. **Не смешивать бота и Zerkalo физически.** Два сервиса, одна экосистема.
6. **Не удалять Личный Миф из публичного интерфейса.** Это вторая дверь.
7. **Не возвращать «Совместимость» в публичный UI.** Пока не готово.
8. **Не копировать канон дословно.** Модель использует его как основу, а не как текст для вставки.
9. **Каждый сильный тезис — с опорой на число.** Без числа нет утверждения.
10. **Голос — один.** Zerkalo, бот, PDF, follow-up — один проводник, разные форматы.
