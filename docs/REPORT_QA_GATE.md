# REPORT QA GATE

> Версия: 1.0
> Дата: 2026-05-01
> Назначение: Обязательный чеклист перед отправкой любого разбора клиенту.

---

## Смысл

- [ ] Текст привязан к конкретным числам из расчёта?
- [ ] Использованы все 5 главных чисел (Душа, Путь, Выражение, Направление, Результат)?
- [ ] Составные числа распакованы там, где это усиливает вывод?
- [ ] Есть главный узор (связка 3 ключевых чисел)?
- [ ] Есть практический маршрут или конкретный шаг?
- [ ] Матрица описана (повторы, дефициты)?
- [ ] Денежный код привязан к financial_code?
- [ ] Каждая секция добавляет новый слой, а не переформулирует предыдущую?

## Тон

- [ ] Не сухо? (есть живые формулировки, текст дышит)
- [ ] Не банально? (нет «вы — творческий и целеустремлённый»)
- [ ] Не инфоцыгански? (нет пустых обещаний, нет давления)
- [ ] Не эзотерично? (нет вибраций, карм, высших сил)
- [ ] Не терапевтически? (нет «позвольте себе», «обнимите тень»)
- [ ] Не фаталистично? (нет «вам предназначено», «судьба неизбежна»)
- [ ] Звучит как живой эксперт, а не как безопасная AI-модель?
- [ ] 80% аналитической точности + 20% тепла?

## Язык

- [ ] Нет грамматических ошибок (падежи, согласования)?
- [ ] Нет машинных фраз («это указывает на потенциал»)?
- [ ] Нет повторов одной мысли разными словами?
- [ ] Нет клише («это говорит о том, что»)?
- [ ] Нет дешёвых усилителей (кайф, пахать, жесть, прокачать)?
- [ ] Нет странных калек (AI-артефактов)?
- [ ] Нет 3-го лица (он, она, этот человек)?
- [ ] Нет конструкций-досье («вы — человек, который»)?
- [ ] Нет декоративных парных контрастов («лёд и огонь», «сталь и шёлк»)?
- [ ] Имя клиента использовано не более 1 раза?

## Safety

- [ ] Нет медицинских диагнозов или рекомендаций?
- [ ] Нет обещаний конкретного финансового результата?
- [ ] Нет предсказаний конкретных событий и дат?
- [ ] Нет рекомендаций прекратить приём лекарств?
- [ ] Нет утверждений, что числа «определяют» судьбу?
- [ ] Нет экстремальных телесных заданий?
- [ ] Есть дисклеймер в конце документа?

## Продукт

- [ ] Понятно, за что человек заплатил?
- [ ] Есть ощущение персональности (не generic-шаблон)?
- [ ] Есть следующий шаг (что делать)?
- [ ] Есть желание вернуться в диалог?
- [ ] Числовой паспорт — сухой и чистый (без трактовок)?
- [ ] Читатель может отличить свой разбор от разбора другого человека?
- [ ] Документ хочется перечитать?

---

## Процедура

1. **Автор** генерирует разбор.
2. **Автор** проходит чеклист выше. Каждый пункт — галочка.
3. Если 3+ пункта непройдены — **доработка перед отправкой**.
4. Если 0–2 пункта непройдены — допустимо отправить с пометкой, что нужно доработать.
5. Первые 20 разборов проходят **ручной QA** полностью.
