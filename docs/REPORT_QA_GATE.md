# REPORT QA GATE — Digital Code / Zerkalo

_Last updated: 2026-04-29_

This checklist is used before delivering any Big Research / Full Mirror document to a client.

## 1. Calculation gate

The report must pass:

- date is correct;
- five main numbers are calculated correctly;
- compound numbers are preserved;
- matrix is calculated correctly;
- personal year/current period is calculated correctly if used;
- no root number is substituted for a compound number in places where the compound matters.

If calculation is uncertain, stop delivery.

## 2. Source discipline gate

The report must distinguish between:

- confirmed canon;
- practice-based interpretation;
- symbolic/editorial bridge;
- hypothesis.

Do not present a new interpretive bridge as canon.

## 3. Structure gate

The report must include:

- cover;
- how to read;
- numeric passport;
- main pattern;
- five main numbers;
- compound numbers;
- matrix;
- strengths;
- tension points;
- money/realization;
- relationships if relevant;
- current period if available;
- 30-day route;
- Living Word;
- next step.

For MVP, some sections may be short, but the skeleton must be visible.

## 4. Personalization gate

The report fails if it could be sent to any person with the same root number.

It must include:

- the exact full formula;
- the person's compound numbers;
- a synthesis across positions;
- at least one precise central pattern;
- practical conclusions specific to this formula.

## 5. Tone gate

The text must be:

- precise;
- alive;
- calm;
- mature;
- premium;
- not manipulative;
- not inflated.

Reject if it sounds like:

- a generic AI report;
- social media motivation;
- occult performance;
- therapy session;
- sales pressure;
- vague praise.

## 6. Language gate

Remove or rewrite:

- repeated phrases;
- weak fillers;
- excessive “deep”, “powerful”, “unique”, “strong”;
- constant “this means that…”;
- awkward machine phrasing;
- grammatical errors;
- inflated claims.

Preferred sentence style:

```text
short recognition -> explanation -> life expression -> practical conclusion.
```

## 7. Safety gate

The report must not contain:

- diagnosis;
- treatment recommendations;
- guaranteed events;
- deterministic predictions;
- fatalistic claims;
- financial guarantees;
- legal advice;
- medical advice;
- claims of healing;
- instructions for unsafe behavior.

## 8. Product value gate

Before delivery, ask:

1. Would the client understand why this is paid?
2. Is there a clear difference from the free First Mirror?
3. Is the document useful after reading, not only beautiful?
4. Is there a next action?
5. Would this create trust for a live consultation or follow-up?

If the answer is no, revise.

## 9. Design gate

For PDF / premium file:

- typography is calm and readable;
- section hierarchy is clear;
- tables are not overloaded;
- page breaks are controlled;
- cover feels premium;
- no decorative noise;
- enough white space;
- the design serves the text.

## 10. Final manual QA checklist

Before sending:

- [ ] Calculation checked.
- [ ] Formula checked.
- [ ] Client name/date checked.
- [ ] All public claims safe.
- [ ] No secret/internal notes included.
- [ ] No accidental placeholders.
- [ ] CTA and next step are clear.
- [ ] PDF opens correctly.
- [ ] Mobile reading acceptable if delivered by Telegram.
- [ ] Final file name is clean.

## 11. Delivery rule

Do not deliver automatically until there is a reliable generator + QA gate.

For first paid smoke-test:

```text
manual generation -> human review -> corrections -> PDF delivery -> follow-up.
```
