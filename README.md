# Zerkalo / «Цифровой Код»

Это премиум-интерфейс инструмента самопознания по системе «Цифровой Код» (развитие системы Канон). Приложение анализирует дату рождения, выстраивает персональную архитектуру (Матрицу и Пять главных чисел) и генерирует «Первое зеркало» — базовый разбор сильных сторон, точек напряжения и практического маршрута.

## Как это устроено

### Interpretation Core
Математическое ядро (`calculator.ts`) переводит дату рождения в 5 главных и 35 составных чисел. Аналитический модуль (`numberKnowledge.ts`, `compoundKnowledge.ts`) сопоставляет эти числа с архетипами, планетами и ключевыми энергиями.

### First Mirror (Первое зеркало)
Главный вау-эффект приложения. Пользователь вводит дату и получает структурированный текст, содержащий: 
- Главный узор (синтез чисел Души и Пути)
- Светлые стороны и зоны напряжения
- Первый практический шаг

Интерфейс выполнен в стиле *modern premium editorial* с элементами античной лаконичности: мрамор, тёплая бумага, античное золото, шрифты с засечками.

### Big Research Teaser & Lead Capture
В конце «Первого зеркала» и «Личного Мифа» пользователю предлагается «Большое исследование» — премиальный персональный документ. В текущей версии для smoke-test этот процесс **полностью ручной**: автоматическая генерация PDF, личный кабинет и автоматическая оплата отключены.

Нажатие на кнопку вызывает модальное окно заявки (`LeadModal`). Заявка отправляется на `/api/lead` и собирается на бэкенде. Далее связь с клиентом устанавливается вручную.

### Примечание к smoke-test
В публичной навигации временно оставлены только два рабочих режима:
1. Архитектура (`CodeArchitecture`)
2. Личный Миф (`PersonalMyth`)

Остальные режимы (`Совместимость` и отдельная `Сказка`) сохранены в кодовой базе, но скрыты из публичной навигации до следующих этапов разработки.

### Локальная детерминированная генерация
Если API недоступно, приложение использует `deterministicMirror` из `interpretation.ts`, собирая структурное Первое Зеркало на основе базы знаний без помощи нейросетей.

### Gemini как опциональный enhancer
При наличии `GEMINI_API_KEY` на сервере Node + Express система отправляет детерминированный каркас и safety-правила в LLM. Модель стилизует текст под глубокий, премиальный, авторский слог и возвращает JSON объект `FirstMirror`.

### Safety & Ethical Boundaries
Важное правило продукта: **проект использует аналитико-образный язык, но не является терапией, диагностикой, предсказанием или духовной практикой.**

Никаких диагнозов, фатальности или магии. Роль приложения — структурно подсветить потенциал, точки напряжения и возможный следующий шаг, а выбор всегда остаётся за пользователем.

## Personal Myth / Сказка про тебя

«Личный миф» — образно-рефлексивный слой проекта. Он помогает увидеть текущее состояние через метафорическую историю, главный образ, внутреннее напряжение, скрытый ресурс и один мягкий следующий шаг.

Это не диагностика, не инструкция к действию и не замена живой поддержке.

## Privacy

Для smoke-test добавлена страница `/privacy.html`. Она описывает, какие данные собираются в формах заявки, зачем они нужны, где хранятся и как пользователь может запросить удаление данных.

```bash
npm install

# Для использования ИИ-генерации текста (опционально)
cp .env.example .env
# Вставьте GEMINI_API_KEY в .env

# Запуск dev-сервера (Frontend + Backend на порту 3000)
npm run dev

# Production-запуск после npm run build
npm run start
```

## Тестирование (How to Test)
Проект покрыт unit-тестами с помощью **Vitest**. Проверяется математика калькулятора, корректность структуры Первого Зеркала, story-layer и отсутствие запрещённых слов в словаре системы.

```bash
npm run lint
npm run build
npm run test
```
