# VYAZEMSKY SKILL 06 — MONEY AND CYCLES

_Status: v0.1 placeholder / requires canon review_
_Last updated: 2026-04-30_

This skill is a placeholder for the missing Skill 06 referenced by the premium file system.

It must not be treated as final canon until reviewed by the founder and aligned with the confirmed calculation methodology.

## 1. Purpose

This skill describes how to speak about money, realization, work rhythm, and life cycles inside premium Digital Code / Zerkalo outputs.

It is used as a support layer for:

- `PREMIUM_FILE`;
- Big Research documents;
- future compatibility interpretation;
- money and realization sections;
- period/cycle reflections when calculation is available.

## 2. Core position

Money is not treated as a prediction.

Safe framing:

```text
money as value creation, rhythm, responsibility, attention, exchange, and practical testing.
```

Unsafe framing:

```text
money as guaranteed fate, fixed income, lucky year, or karmic punishment.
```

## 3. Allowed use

Allowed:

- describe earning style as an analytical hypothesis;
- name possible value creation modes;
- show where energy or attention may leak;
- suggest practical experiments;
- connect work rhythm to the person’s formula;
- discuss cycles as focus themes, not fixed events.

## 4. Forbidden use

Forbidden:

- guaranteed income claims;
- investment advice;
- exact financial predictions;
- “you will become rich if…”;
- “this year guarantees money”;
- fatalistic statements about poverty or success;
- fear-based karmic explanations;
- instructions that replace professional financial advice.

## 5. Safe usage formula

Use this formula when writing money sections:

```text
formula source -> value creation mode -> tension/leak -> practical experiment -> boundary statement.
```

Example pattern:

```text
This formula may create value where structure, explanation, and trust are needed. The leak appears when attention spreads across too many unfinished forms. The useful experiment is to test one paid format for 30 days, not to build a complete system before the first client.
```

## 6. Cycles boundary

Cycles may be used as a reflective navigation layer only when calculation is available.

Correct:

```text
This period may emphasize the theme of structure and responsibility. It is useful to test one concrete form.
```

Incorrect:

```text
This period will bring money and success.
```

## 7. Relation to PREMIUM_FILE

In premium files, this skill should support:

- money and realization section;
- 30-day practical route;
- current period commentary;
- founder/operator recommendations;
- future follow-up prompts.

It should not dominate the report or turn it into a business forecast.

## 8. Relation to COMPATIBILITY

For future compatibility modes, money/cycles may be mentioned only as interaction dynamics:

- different work rhythms;
- different risk tolerance;
- different pace of decision-making;
- different value exchange styles.

Do not predict whether a couple or partnership will become rich or fail.
